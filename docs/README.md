# Docs

Structured project documentation.

## Sections

- `overview/`: high-level project overview and entry documentation
- `architecture/`: system architecture notes and diagrams
- `reference/`: API and contract reference material
- `design/`: UI and design language documentation
- `theory/`: conceptual and mathematical framing
- `planning/`: active planning and task-tracking documents
- `tutorials/`: notebooks and guided walkthroughs
