Slide 02 ELI5

What Phase 2 changed, in three pictures.

Phase 1 gave us a librarian, a weatherman, and an advisor. Phase 2 doesn't invent new people — it builds the office they all work in.

Filing cabinet (2C)

Every historical weather window used to be scribbled on whatever sticky note a person had at hand. Now there is a locked filing cabinet with 27 labeled folders: calm, crisis, trend, rangebound. Every benchmark reads from the same folders so yesterday's result and tomorrow's result are directly comparable.

Report template (2B)

Every bench used to write its own memo in its own voice. Now there is one printed form: here is what I tested, here is the baseline number, here is the candidate number, here are the pass/fail boxes, here is why I think it passed or failed. Different benches, identical form. Plus a rejection drawer — every idea we killed goes in with "what would make us revisit this."

Outside opinion desk (2A)

Until now we only compared the engine to itself. Now there's a desk where standard foundation-model "outside opinions" sit down for the same test under the same form. Most of those opinions couldn't get to work today (their weights wouldn't download in this environment), so the office is built but mostly empty. Next sprint they show up for real.

The point: Phase 1 was "what did we measure?" Phase 2 is "how will every future measurement be trustworthy and comparable?" The unglamorous work that makes the glamorous work honest.

The invariant that matters: append-only. A slice ID that lands on main can never be renamed or deleted. Every historical ledger row references these IDs; renaming would silently invalidate our backtest record.

The validator bug

First-session validate.py captured module constants in function defaults. Python evaluates defaults at def-time, so monkeypatch in tests did nothing. Fix: late-bind. Worth remembering.

Known gaps

No 2008 GFC (dataset won't go back). No equity-vs-rates pair. Mean-reverting crypto is thin (one slice). These aren't blockers for Phase 3 but they're documented.

Migrations landed

1A's retrieval_bench/slices.yaml and 1B's projector-v2 benchmark YAML both reference catalogue IDs now. 1A tests still green post-migration.

Slide 05 Finding 5 — Autoresearch core Infra / Live

One ledger. One gate vocabulary. One report template. One rejection drawer.

Five modules under research/autoresearch/core/:

  • ledger.py — canonical LedgerEntry; append, query, compare
  • metrics_delta.py — paired bootstrap (n=1000, seed=42), returns significance
  • gates.py — declarative Gate(name, metric, threshold, direction, required)
  • report.py — canonical markdown template every lane renders through
  • rejection_log.py — appender for rejections.jsonl with revisit_conditions
The revisit_conditions field is the interesting idea. Every killed direction has a written condition under which we would re-test it. Six months from now, an agent reading the log can mechanically answer "has the world changed enough to revisit this?" It forces the killer to commit to what would change their mind.

Backfilled rejections

tier2_as_default (preliminary, 2026-04-15) — revisit if NVDA/TSLA/BTC + seed=314 show any regime where Tier 2 improves CRPS.

regime_aware_widening (hard, 2026-04-15) — revisit if someone refits the multipliers against real residuals.

Why declarative gates

Hand-written gate logic in each lane doesn't scale past three lanes. Declarative gates make cross-lane comparison mechanical, and two agents reviewing the same bench arrive at the same verdict.

Slide 07 Why the wavelet number matters anyway

A classical baseline we must actually clear.

The plan's Rule #4 is benchmark before belief. Before we claim "our engine does something no standard model does," we need an honest comparison. With the foundation-model cells down, the only remaining comparison is the classical wavelet baseline:

0.48
wavelet hit rate (best in sweep)
0.1083
wavelet mean CRPS
0.279
wavelet calibration error

The engine (Tier 1 only, adaptive-conformal projector) needs to beat this on the same 4 slices. That's the next sprint's first bench.

A classical baseline that a beginner could implement in a weekend is more useful to us than an unreachable TimesFM number. If we can't beat wavelet + AR(p), we're not ready to pitch foundation-model superiority.
Slide 08 Synthesis

Phase 2 is infrastructure. The verdicts are Phase 3's job.

Every future bench now has:

  1. A locked set of regime-labeled slices to run against.
  2. A canonical ledger schema, gate vocabulary, and report template.
  3. A rejection log that forces "what would make us revisit this?" on every killed direction.
  4. A classical baseline (wavelet) to clear before any external claim.
  5. An adapter registry ready to absorb real foundation-model weights the moment they're reachable.

Phase 1 was the first real measurements. Phase 2 is the office those measurements now live in. Phase 3's benches will be cheaper and harder to fool than Phase 1's were.

Slide 09 Still open

What we still don't know.

  • Real TimesFM / Chronos / Moirai / MOMENT numbers on our slices.
  • Whether the engine (Tier 1 + adaptive conformal) beats the wavelet classical baseline.
  • Whether the gate layer has a bug — only 8 happy-path + edge-case tests today; gate bugs silently flip every bench verdict and deserve property-based testing.
  • Whether the budget-expanded retrieval rerun (NVDA, TSLA, BTC, seed=314) flips or confirms the tier2-as-default verdict.
  • Whether change-aware conformal actually diverges from plain adaptive on a shift-rich slice.
  • Adaptive conformal on real parquets (not synthetic fallback). Second confirmation is needed before promoting to default.
Slide 10 What Phase 3 needs to do

Reuse everything. Integrate via seams, not replacements.

Reuse the infra

The JEPA / world-model lane should emit canonical ledger rows through core.ledger, score against canonical core.gates, render through core.report, and use catalogue slice IDs. Zero new scaffolding.

Don't replace the matcher

Phase 1 already established the matcher as well-understood (and Tier 2 as the suspect layer). The world model enters as a projector variant or a confidence-signal source, not as a replacement for retrieval.

The decision rule for Phase 3 promotion should mirror 1B's: a world-model component promotes to default only if it improves CRPS, calibration, or useful confidence behavior — on canonical slices, through canonical gates, with a ledger row an auditor can read.
